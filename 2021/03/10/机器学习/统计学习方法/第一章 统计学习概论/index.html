第一章 统计学习及监督学习概论

统计学习

统计学习定义

基于数据构建概率统计模型并运用模型对数据进行预测

统计学习的对象

统计学习的研究对象是数据，挖掘隐藏在数据中的信息。数据包括数字、文字、图像、视频、音频以及它们的组合。

统计学习的目的

对未知数据的预测和分析。

统计学习的方法

基于数据构建统计模型对数据进行预测和分析。统计学习由监督学习、无监督学习和强化学习组成。

统计学习步骤

  1. 得到一个有限的训练数据集合
  2. 确定假设空间
  3. 确定学习的策略
  4. 确定学习的算法
  5. 通过学习选择最优模型
  6. 利用模型预测数据

统计学习的分类

基本分类

  • 监督学习：从标注数据中心学习预测模型
  • 无监督学习：从无标注数据中心学习预测模型
  • 强化学习：智能系统在与环境的连续互动中学习最优行为策略
  • 半监督学习：通过少量标注数据和大量未标注数据学习预测模型

按模型分类

  • 概率与非概率模型
  • 线性模型与非线性模型
  • 参数化与非参数化模型

按算法分类

  • 在线学习：每次接受一个样本，进行预测之后学习模型，之后不断重复该操作的机器学习
  • 批量学习：学习一次接受所有数据，学习模型，之后进行预测

按技巧分类

  • 贝叶斯学习：利用贝叶斯定理，计算在给定条件下模型的条件概率，即后验概率
  • 核方法：使用核函数表示和学习非线性模型

统计学习方法三要素

方法 = 模型 + 策略 + 算法

模型

模型是要学习的条件概率分布获决策函数，可以定义为决策函数的集合。

  • 决策函数的集合（非概率模型）
  • 条件概率的集合（概率模型）

策略

统计学习的目标是从假设空间中选取最优模型。

损失函数和风险函数

损失函数用来评价模型的预测值和真实值不一样的程度，损失函数越小，通常模型的性能越好。不同的模型用的损失函数一般也不一样。

损失函数分为经验风险损失函数和结构风险损失函数。经验风险损失函数指预测结果和实际结果的差别，结构风险损失函数是指经验风险损失函数加上正则项。

常用的损失函数有以下几种

  • 0-1损失函数（zero-one loss）

特点：直接对应分类错误的个数，但它是一个非凸函数，不常用。感知机就是用的这种损失函数。

  • 平方损失函数（quadratic loss）

特点：经常用于回归问题

  • 绝对损失函数（absolute loss）
  • 对数损失函数（logarithmic loss）

特点：

  1. log对数损失函数能非常好的表征概率分布，在很多场景尤其是多分类，如果需要知道结果属于每个类别的置信度，那它非常适合。
  2. 健壮性不强，相比于hinge loss对噪声更敏感。
  3. 逻辑回归的损失函数就是log对数损失函数。
  • 指数损失函数（exponential loss）

特点：对离群点、噪声非常敏感。经常用在AdaBoost算法中。

  • Hinge损失函数

特点：

  1. hinge损失函数表示如果被分类正确，损失为0，否则损失就为1-y f(x)。SVM就是使用这个损失函数。
  2. 健壮性相对较高，对异常点、噪声不敏感，但它没太好的概率解释。
  • 交叉熵损失函数 (Cross-entropy loss function)

特点：

  1. 本质上也是一种对数似然函数，可用于二分类和多分类任务中。
  2. 当使用sigmoid作为激活函数的时候，常用交叉熵损失函数而不用均方误差损失函数，因为它可以完美解决平方损失函数权重更新过慢的问题，具有“误差大的时候，权重更新快；误差小的时候，权重更新慢”的良好性质。

经验风险最小化与结构风险最小化

  • 经验风险最小化

     经验风险最小化策略认为，经验风险最小的模型就是最优的模型，即：

    当样本容量足够大时，经验风险最小化能保证具有较好的学习效果，在现实中也被广泛采用。例如极大似然估计就是经验风险最小化的一个例子（在模型是条件概率分布，损失函数是对数损失函数时等价）。

  • 结构风险最小化

    当样本容量较小时，经验风险最小化的准则会导致过拟合问题的出现，结构风险最小化等价于正则化（也叫做罚项）。定义如下：

    结构风险较小的模型往往对训练数据以及未知的测试数据都有较好的预测。例如贝叶斯估计中的最大后验概率估计就是结构风险最小化的例子（在模型是条件概率分布，损失函数是对数损失概率且模型复杂度由模型的先验概率表示时等价）。

算法

算法是指学习模型的具体计算方法。统计学习基于训练数据集，根据学习策略, 从假设空间中选择最优模型，最后需要考虑用什么样的计算方法求解最优模型。

模型评估与选择

数据集的划分

一般将数据集划分为三部分：

  • 训练集：训练模型，迭代优化模型的参数
  • 验证集：调整超参数，优化模型
  • 测试集：对模型的性能进行评估（精度或泛化能力）

训练误差和测试误差

  • 训练误差：也称作经验误差，指在训练集上的误差
  • 测试误差：在测试集上的误差，用来衡量模型的泛化能力

混淆矩阵

混淆矩阵用于二分类问题，可将样例根据其真实类别与学习器预测类别的组合划分为真正例(true positive) 假正例 (false positive) 、真反例(true negative)假反例 (false negative) 四种情形，令 TP FP TN FN 分别表示其对应的样例数，则显然有 TP+FP+TN+FN=样例总数

根据混淆矩阵得到下列评价指标

  • 召回率（Recall）: 也叫查全率，指在实际为正例的样本中被预测为正样本的比例，它的含义类似于宁可错杀一千也不放过一个。
  • 精确率（Precision）:也叫查准率，指预测为正例的样本中实际为正样本的比例
  • 准确率（Accuracy）：指预测为正确的样本占总样本的比列
  • F1值：P和R指标有时候会出现的矛盾的情况，这样就需要综合考虑他们。F1是Precision和Recall加权调和平均：

偏差和方差

  • 偏差：期望和实际的差值
  • 方差：代表模型的稳定性，可以衡量模型的泛化能力
  • 实际误差：$\operatorname{Err}(\mathrm{x})=$ Bias $^{2}+$ Variance $+$ Irreducible Error

一般模型的越复杂，其拟合能力就越强，偏差就越低，但也越容易过拟合，导致泛化能力下降，模型被当前的数据带偏，导致偏差越高。

过拟合和欠拟合

过拟合可以对应上图曲线交点的右半边，欠拟合对应左半边

欠拟合：

  • 解释：未能很好的拟合数据导致预测能力不足
  • 实际中表现为：训练误差高，测试误差高
  • 原因：模型太简单，训练迭代次数太少
  • 解决办法：换更复杂的模型训练，增加训练的迭代次数

过拟合：

  • 解释：过度拟合数据导致的模型泛化能力降低
  • 实际中表现为：训练误差低，测试误差高
  • 原因：数据太少或模型太复杂
  • 解决办法：
    1. 增加数据（数据增强）
    2. 降低特征的维度，对于一些特征工程而言，可以降低特征的数量——删除冗余特征，人工选择保留哪些特征。这种方法也可以解决过拟合问题。
    3. 降低模型复杂度或使用正则化（L1正则、L2正则），正则化的作用是选择训练误差和模型复杂度同时较小的模型。
    4. 提前终止训练

正则化与交叉验证

正则化和交叉验证是常用的模型选择方法

正则化

正则化是在经验风险上加一个正则化项（regularization）或罚项（penalty term），是选择训练误差和模型复杂度同时较小的模型。正则化一般具有如下形式：

正则化项可以有不同的形式：

  • 参数向量的L1范数：参数的绝对值之和
  • 参数向量的L2范数：参数的平方和再开方

为什么正则化可以可以避免过拟合呢？

一般模型过拟合的原因是因为模型的参数过多，如果可以将模型的一些参数趋于0或者等于0，那么就可以降低模型的复杂度。正则化的目的就是这个，即给风险函数加上一个约束条件，限制风险函数中参数的取值。

想象现在只有两个参数w1 w2 要学，蓝色的圆心是误差最小的地方，而每条蓝线上的误差都是一样的。正则化的方程是在红线上产生的额外误差(也能理解为惩罚度)，在红圈上的额外误差也是一样。所以在蓝线和红线交点上的点能让两个误差的合最小。这就是 w1 和 w2 正则化后的解。要提到另外一点是，使用 L1 的方法，我们很可能得到的结果是只有w2的特征被保留，使得L1正则具有稀疏性，所以很多人也用 L1 正则化来挑选对结果贡献最大的重要特征。但是 L1 的结果并不是稳定的。比如用批数据训练，每次批数据都会有稍稍不同的误差曲线。

交叉验证

在数据充足的时候，可以简单的将数据划分为训练集、验证集、测试集，但是在许多实际的应用中数据是不充足的。为了选择更好的模型可以采用交叉验证方法。交叉验证的思想是重复的使用数据；把给定的数据进行切分，将切分的数据集组合为训练集和测试集，在此基础上反复的训练、测试以及模型的选择。

  • 简单交叉验证
  • S折交叉验证
  • 留一交叉验证

生成模型和判别模型

判别模型和生成模型的区别如下：

可见，生成模型多了生成正、负类分布的过程，而这个过程就是在尝试学习这些数据到底是怎么生成的，或者说在尝试学习真正的知识。可以这么比喻，判别模型就是不断刷题，不太去理解，这样也可以很好地应付考试（预测）；而生成模型在刷题同时还会尝试理解其中的知识，只要理解得当，完全可以考出好的成绩

从上面的比喻出发，可以进一步理解两者的优劣：

​ （1）刷得题够多，考试成绩就会很好；但如果有些题型没有刷到就会束手无策。也就是说，判别模型只要数据量足够就有很好的泛化能力，但如果遇到没有出现过的情况，那么是无法解决的，这样的例子后面会看到。

​ （2）理解如果出错，考试反而糟糕；但是如果能够正确理解，那么在刷题量不够的情况下，也可以举一反三，甚至可以解决没有遇到过的题型。也就是说，数据量少的时候，生成模型可能有奇效，比如地震数据很少，要预测地震的话或许应该用生成模型。
